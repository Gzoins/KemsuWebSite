# KemGU Backend API v2.0

Современный backend для образовательной платформы КемГУ с аутентификацией, ролями и файловой системой.

## 🚀 Возможности

- **Аутентификация и авторизация** - JWT токены, хеширование паролей
- **Система ролей** - admin, teacher, student с разными правами доступа
- **Управление контентом** - лекции, задания, сабмиты студентов
- **Файловая система** - загрузка и хранение файлов студентами
- **SQLite база данных** - легковесная и надежная
- **REST API** - современный API с полной документацией

## 📋 Требования

- Node.js 16+
- npm или yarn

## 🛠️ Установка

1. Установите зависимости:
```bash
cd server
npm install
```

2. Настройте переменные окружения:
```bash
cp .env.example .env
# Отредактируйте .env файл с вашими настройками
```

3. Запустите миграцию базы данных:
```bash
node database/migrate.js
```

4. Запустите сервер:
```bash
npm start
```

Сервер будет доступен на `http://localhost:4000`

## 📚 API Эндпоинты

### Аутентификация
- `POST /api/auth/register` - Регистрация пользователя
- `POST /api/auth/login` - Авторизация пользователя
- `GET /api/auth/me` - Получение профиля
- `PUT /api/auth/profile` - Обновление профиля
- `PUT /api/auth/change-password` - Смена пароля

### Администрирование
- `GET /api/admin/users` - Список пользователей
- `POST /api/admin/users` - Создание пользователя
- `PUT /api/admin/users/:id` - Обновление пользователя
- `DELETE /api/admin/users/:id` - Удаление пользователя
- `GET /api/admin/stats` - Статистика

### Лекции
- `GET /api/lectures` - Список лекций
- `POST /api/lectures` - Создание лекции
- `PUT /api/lectures/:id` - Обновление лекции
- `DELETE /api/lectures/:id` - Удаление лекции

### Задания
- `GET /api/assignments` - Список заданий
- `POST /api/assignments` - Создание задания
- `PUT /api/assignments/:id` - Обновление задания
- `DELETE /api/assignments/:id` - Удаление задания

### Сабмиты
- `GET /api/submissions` - Список сабмитов
- `POST /api/submissions` - Загрузка работы
- `PUT /api/submissions/:id` - Оценка работы
- `DELETE /api/submissions/:id` - Удаление сабмита

## 🔒 Система ролей

### Admin (Администратор)
- Полный доступ ко всем данным
- Управление пользователями
- Просмотр статистики

### Teacher (Преподаватель)
- Создание и редактирование лекций
- Создание и редактирование заданий
- Просмотр и оценка сабмитов своих заданий
- Просмотр студентов своей группы

### Student (Студент)
- Просмотр лекций своей группы
- Просмотр заданий своей группы
- Загрузка работ на задания
- Просмотр своих оценок

## 📁 Структура проекта

```
server/
├── database/
│   ├── schema.sql      # SQL схема базы данных
│   ├── migrate.js      # Скрипт миграции
│   └── db.js          # Подключение к базе данных
├── middleware/
│   └── auth.js        # Middleware аутентификации
├── routes/
│   ├── auth.js        # Маршруты аутентификации
│   ├── admin.js       # Маршруты администратора
│   ├── lectures.js    # Маршруты лекций
│   ├── assignments.js # Маршруты заданий
│   └── submissions.js # Маршруты сабмитов
├── .env.example       # Пример переменных окружения
├── package.json       # Зависимости
└── server.js          # Главный файл сервера
```

## 🔧 Конфигурация

Основные переменные окружения:

- `PORT` - Порт сервера (по умолчанию 4000)
- `DB_PATH` - Путь к базе данных SQLite
- `JWT_SECRET` - Секрет для JWT токенов
- `JWT_EXPIRES_IN` - Время жизни токена
- `UPLOAD_DIR` - Папка для загрузки файлов

## 🚨 Безопасность

- Пароли хешируются с помощью bcrypt
- JWT токены с ограниченным временем жизни
- Проверка ролей для всех защищенных маршрутов
- Валидация входных данных
- Ограничение размера загружаемых файлов
